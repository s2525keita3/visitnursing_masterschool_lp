# ランディングページ（LP）要件定義書

## 1. プロジェクト概要

### 1.1 プロジェクト名
オンライン起業塾マスタープログラム - 訪問看護ステーション開業支援LP

### 1.2 目的
訪問看護ステーションの開業準備・経営支援サービスへの無料セミナー申込を促進するランディングページ

### 1.3 対象ユーザー
- 訪問看護ステーション開業を検討している看護師
- 訪問看護ステーション経営者
- 開業準備中の方

### 1.4 主要KPI
- 無料セミナー申込数
- 個別相談申込数
- コンバージョン率（CVR）

### 3.2 セクション詳細

#### 3.2.1 Heroセクション
- **目的**: ファーストビューのインパクト最大化
- **要素**:
  - 背景画像（全画面表示）
  - ターゲットテキスト
  - メインタイトル（2行構成、オレンジ強調）
  - サブタイトル
  - 特典・オファー（白背景＋緑ボーダー）
  - CTAボタン（オレンジ、shakeアニメーション）
- **アニメーション**: フェードイン（段階的）

#### 3.2.2 Problemsセクション
- **目的**: 課題の共感を促す
- **要素**:
  - 背景画像（`nayami.png`）
  - メインタイトル（オレンジ色）
  - 6つの問題リスト（カード形式）
  - CTAボタン
- **レイアウト**: 中央配置、カード形式

#### 3.2.3 Benefitsセクション
- **目的**: 解決策と価値を提示
- **要素**:
  - メインタイトル（「5つのベネフィット」）
  - 5つのベネフィットカード（画像＋説明）
  - ソーシャルプルーフ（「98%の融資承認率」）
  - 受講生の成果（3名の動画＋成果データ）
- **レイアウト**: グリッド（3列）

#### 3.2.4 ProgramHighlightsセクション
- **目的**: プログラムの詳細を提示
- **要素**:
  - 6ヶ月間のロードマップ（3つのPhase）
  - 各Phaseの詳細（課題・解決策・武器）
  - 修了証書セクション
- **レイアウト**: カード形式、タイムライン風

#### 3.2.5 Seminarセクション
- **目的**: 無料セミナーの案内
- **要素**:
  - セミナータイトル
  - 4つの講義テーマ
  - オンラインセミナー日程
  - 特典画像（`present.png`）
- **レイアウト**: 中央配置

#### 3.2.6 ContactFormセクション
- **目的**: セミナー申込・個別相談の受付
- **要素**:
  - お名前（必須）
  - メールアドレス（必須）
  - お申し込み種別（セミナー申込/個別相談）
  - 希望セミナー日程（セミナー申込時のみ）
  - ご相談内容（任意）
  - 送信ボタン
- **バリデーション**: リアルタイム＋送信時
- **送信先**: Formspree

## 4. デザインシステム

### 4.1 カラーパレット

#### ブランドカラー
- **オレンジ（Primary）**: `#FF6B00` / `brand-orange`
- **ダーク（Text）**: `#000000` / `brand-dark`
- **ミュート（Secondary Text）**: `#666666` / `brand-muted`
- **ライト（Background）**: `#FFF8F0` / `brand-light`
- **クリーム（Background）**: `#FFF8F0` / `bg-cream`

#### アクセントカラー
- **エメラルド（Success/CTA）**: `#06C755` / `emerald-500`
- **レッド（Error）**: `#DC2626` / `red-500`, `red-600`
- **イエロー（Highlight）**: `#FCD34D` / `yellow-300`

### 4.2 タイポグラフィ

#### フォントファミリー
- **メインフォント**: Noto Sans JP
- **フォールバック**: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif

#### フォントサイズ
- **メインタイトル**: `text-4xl md:text-5xl lg:text-6xl` (36px → 48px → 60px)
- **セクションタイトル**: `text-3xl md:text-4xl lg:text-5xl` (30px → 36px → 48px)
- **本文**: `text-base md:text-lg` (16px → 18px)
- **小見出し**: `text-xl md:text-2xl` (20px → 24px)

#### フォントウェイト
- **見出し**: `font-black` (900)
- **強調**: `font-bold` (700)
- **本文**: `font-medium` (500) / `font-semibold` (600)

### 4.3 スペーシング

#### セクションパディング
- **モバイル**: `py-20` (80px)
- **タブレット**: `sm:py-24` (96px)
- **デスクトップ**: `md:py-32 lg:py-40` (128px → 160px)

#### 要素間隔
- **カード間**: `gap-6 md:gap-8` (24px → 32px)
- **コンテンツ内**: `space-y-4 md:space-y-6` (16px → 24px)

### 4.4 シャドウ

#### カードシャドウ
- **通常**: `shadow-card` (0 8px 32px rgba(0, 0, 0, 0.12))
- **ホバー**: `shadow-card-hover` (0 16px 48px rgba(0, 0, 0, 0.16))

#### CTAシャドウ
- **通常**: `shadow-cta` (0 12px 32px rgba(255, 107, 0, 0.35))
- **ホバー**: `shadow-cta-hover` (0 16px 40px rgba(255, 107, 0, 0.45))

### 4.5 ボーダー

#### 角丸
- **カード**: `rounded-3xl` (24px)
- **ボタン**: `rounded-[50px]` (50px、完全な円形)
- **入力フィールド**: `rounded-2xl` (16px)

#### ボーダー幅
- **通常**: `border` (1px)
- **強調**: `border-2` (2px)
- **特典ボックス**: `border-[3px]` (3px)

### 4.6 アニメーション

#### フェードイン
- **fade-up**: 下から上にフェードイン（40px移動）
- **fade-in**: 単純フェードイン
- **slide-in-left**: 左からスライドイン
- **slide-in-right**: 右からスライドイン

#### 特殊アニメーション
- **shake-slow**: CTAボタンのゆっくりした揺れ（2秒間隔）
- **animate-pulse**: パルスアニメーション

#### アニメーション遅延
- **ベース**: 0.1秒
- **段階的**: 0.1秒間隔で遅延
- **Hero**: 0.1秒 → 0.2秒 → 0.25秒 → 0.4秒

## 5. コンポーネント仕様

### 5.1 共通コンポーネント

#### Section
- **用途**: セクションラッパー
- **Props**: `id`, `tone` (surface/light/dark)
- **パディング**: レスポンシブ対応

#### Card
- **用途**: カードレイアウト
- **Props**: `children`, `className`, `padding` (sm/md/lg), `hover`, `style`
- **スタイル**: 白背景、シャドウ、ホバー効果

#### CTAButton
- **用途**: コールトゥアクションボタン
- **Props**: `href`, `onClick`, `size` (small/default/large), `isLoading`
- **スタイル**: オレンジグラデーション、シャドウ、shakeアニメーション

#### ImageWithFallback
- **用途**: 画像表示（フォールバック付き）
- **Props**: `src`, `alt`, `className`, `fallbackText`, `priority`
- **機能**: 画像読み込み失敗時のフォールバック表示

### 5.2 レイアウトコンポーネント

#### Header
- **固定位置**: `sticky top-0`
- **要素**: 組織名、CTAボタン
- **スクロール時**: 背景色・シャドウ変更

#### Footer
- **要素**: 会社情報、所在地、事業内容、連絡先（メール、公式LINE）
- **レイアウト**: 4カラムグリッド

#### StickyBottomCTA
- **固定位置**: `fixed bottom-0`
- **表示条件**: フッター手前200px以内で非表示
- **要素**: メッセージ＋CTAボタン

#### ProgressBar
- **位置**: 画面最上部
- **機能**: スクロール進捗を視覚化

### 5.3 機能コンポーネント

#### ContactForm
- **フォーム管理**: `useForm`フック
- **バリデーション**: `validateContactForm`関数
- **送信先**: Formspreeエンドポイント
- **状態管理**: idle → validating → submitting → success/error

## 7. パフォーマンス最適化

### 7.1 コード分割
- **Hero**: 通常ロード（ファーストビュー）
- **その他**: 遅延読み込み（`lazy`）

### 7.2 画像最適化
- **優先画像**: `priority={true}`（Hero背景画像）
- **遅延読み込み**: `priority={false}`（その他）
- **フォールバック**: 画像読み込み失敗時の代替表示

### 7.3 スクロール最適化
- **統合リスナー**: `useScrollListener`フックで複数コンポーネントのスクロールイベントを統合
- **Throttle**: 16ms（60fps）でthrottle

### 7.4 ビルド最適化
- **チャンク分割**: vendor、iconsを分離
- **ミニファイ**: esbuild使用
- **console.log削除**: 本番環境で自動削除

## 8. アクセシビリティ

### 8.1 WCAG 2.1準拠
- **タッチターゲット**: 最小44px（推奨48px）
- **コントラスト**: テキストと背景のコントラスト比4.5:1以上
- **キーボードナビゲーション**: 全要素がキーボードで操作可能

### 8.2 ARIA属性
- **aria-label**: すべてのインタラクティブ要素
- **aria-invalid**: フォームエラー状態
- **aria-describedby**: エラーメッセージとの関連付け
- **role**: alert、progressbarなど

```

├── features/           # 機能別コンポーネント
│   ├── hero/
```
