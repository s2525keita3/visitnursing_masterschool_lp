# プロジェクト構造リファクタリングガイド

## 概要

このスクリプトは、現在のフラットなコンポーネント構造を機能ベースの構造に再編成します。

## 使用方法

### 1. ドライラン（推奨：まず実行）

実際にファイルを移動せず、何が起こるかを確認します：

```bash
python reorganize.py --dry-run
```

### 2. 実際の実行

ドライランの結果を確認後、実際にリファクタリングを実行：

```bash
python reorganize.py
```

## 実行される変更

### ファイル移動

- **機能コンポーネント** → `src/features/{feature-name}/`
- **共通UIコンポーネント** → `src/shared/ui/`
- **レイアウトコンポーネント** → `src/shared/layout/`

### import文の自動更新

- `App.tsx`内のすべてのimportパス
- 機能コンポーネント内の共通UIコンポーネント参照

## 安全機能

1. **競合検出**: 同名ファイルが既に存在する場合、タイムスタンプ付きバックアップ名を使用
2. **ログ出力**: すべての操作をJSON形式でログファイルに記録
3. **ドライラン**: 実際の変更前に影響範囲を確認可能

## ログファイル

実行後、`reorganize_log_YYYYMMDD_HHMMSS.json` が生成されます。

## 注意事項

- **Gitコミット推奨**: 実行前に変更をコミットしておくことを推奨します
- **バックアップ**: 重要な変更がある場合は、事前にバックアップを取ってください
- **テスト**: リファクタリング後、アプリケーションが正常に動作するか確認してください
